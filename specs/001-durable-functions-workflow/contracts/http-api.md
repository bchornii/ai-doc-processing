# HTTP API Contract: Document Processing Triggers

**Feature**: `001-durable-functions-workflow`
**Date**: 2026-06-30

---

## HTTP Trigger: Start Document Batch Processing

### Request

```
POST /api/process-documents
Content-Type: application/json
X-Correlation-Id: {uuid}          (optional; generated if absent)
```

**Request Body**:
```json
{
  "container_name": "documents"
}
```

| Field | Type | Required | Constraints |
|---|---|---|---|
| `container_name` | string | Yes | Non-null, non-empty |

**Headers**:
| Header | Type | Required | Description |
|---|---|---|---|
| `X-Correlation-Id` | UUID string | No | Caller-supplied correlation ID. Auto-generated if absent. Propagated through the entire orchestration. |

---

### Response: 202 Accepted (Success)

Returned when a valid request body is received and the root batch orchestration has been scheduled.

```
HTTP/1.1 202 Accepted
Content-Type: application/json
```

**Response Body** (Durable Functions management URLs):
```json
{
  "id": "abc123def456",
  "purgeHistoryDeleteUri": "http://localhost:7071/runtime/webhooks/durabletask/instances/abc123def456?code=...",
  "sendEventPostUri": "http://localhost:7071/runtime/webhooks/durabletask/instances/abc123def456/raiseEvent/{eventName}?code=...",
  "statusQueryGetUri": "http://localhost:7071/runtime/webhooks/durabletask/instances/abc123def456?code=...",
  "terminatePostUri": "http://localhost:7071/runtime/webhooks/durabletask/instances/abc123def456/terminate?reason={text}&code=..."
}
```

This response body is generated by the Durable Functions runtime via `DurableTaskClient.CreateCheckStatusResponseAsync()`.

**Polling the status URL** (`statusQueryGetUri`):
```json
{
  "name": "ProcessDocumentBatchWorkflow",
  "instanceId": "abc123def456",
  "runtimeStatus": "Completed",
  "input": { "containerName": "documents", "correlationId": "..." },
  "output": {
    "messages": ["Processed folder1/doc1.pdf successfully", "Processed folder2/doc2.pdf successfully"],
    "errors": []
  },
  "createdTime": "2026-06-30T10:00:00Z",
  "lastUpdatedTime": "2026-06-30T10:00:05Z"
}
```

---

### Response: 400 Bad Request (Validation Failure)

Returned when the request body cannot be deserialized or `container_name` is missing or empty. No orchestration is started.

```
HTTP/1.1 400 Bad Request
Content-Type: application/problem+json
```

**Response Body** (RFC 9457 Problem Details):
```json
{
  "title": "Invalid Request",
  "status": 400,
  "detail": "The 'container_name' field is required and must not be empty."
}
```

| Field | Value |
|---|---|
| `title` | Human-readable error category |
| `status` | `400` |
| `detail` | Specific description of the validation failure |

## Queue Trigger: Automated Batch Processing

### Message Format

**Queue name**: `documents`
**Encoding**: Base64-encoded UTF-8 JSON

**Decoded message body**:
```json
{
  "container_name": "documents"
}
```

The decoded schema is identical to the HTTP trigger request body.

### Success Behavior

A valid, decodable message triggers the same `ProcessDocumentBatchWorkflow` root orchestration as the HTTP trigger. No response is returned (fire-and-forget from the queue's perspective).

### Failure Behavior

| Failure Scenario | Behavior |
|---|---|
| Message cannot be Base64-decoded | Dead-lettered after retry exhaustion; error logged with correlation ID |
| Decoded JSON cannot be deserialized to `DocumentBatchRequest` | Dead-lettered after retry exhaustion; error logged |
| `container_name` missing or empty | Dead-lettered after retry exhaustion; error logged |

## Orchestration Management API

The Durable Functions runtime exposes the following management endpoints automatically (via `DurableTaskClient.CreateCheckStatusResponseAsync`):

| Method | Path | Description |
|---|---|---|
| `GET` | `/runtime/webhooks/durabletask/instances/{instanceId}` | Query orchestration status |
| `POST` | `/runtime/webhooks/durabletask/instances/{instanceId}/raiseEvent/{eventName}` | Send external event |
| `POST` | `/runtime/webhooks/durabletask/instances/{instanceId}/terminate` | Terminate orchestration |
| `DELETE` | `/runtime/webhooks/durabletask/instances/{instanceId}` | Purge orchestration history |

**Status values**: `Pending`, `Running`, `Completed`, `Failed`, `Terminated`

**404 response**: If an unknown `instanceId` is queried, the runtime returns HTTP 404.
